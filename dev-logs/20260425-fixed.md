# 2026-04-25 V2 Logging Storage Update

## Problem

The first three live days produced about 32.3 MB of copied logs under:

```text
RDP/logs/portfolio_option_a_202604/
```

The largest files were:

```text
events_202604.csv   ~15.0 MB
signals_202604.csv  ~12.0 MB
timing_202604.csv   ~3.1 MB
snapshot_202604.csv ~2.0 MB
```

Most of the waste came from repetitive per-loop rows that carried no new trading
information:

```text
SYMBOL_SKIPPED_NO_NEW_BAR
signals rows with eval_status=not_due
SNAPSHOT_OK event rows duplicating snapshot.csv
```

## Decision

Keep these logs unchanged as the primary trading truth:

```text
trades
reducer
snapshot
state_audit
```

Thin the noisy logs:

```text
events  -> log state change / meaningful runtime transitions only
signals -> log only real evaluations, not not_due rows
```

## Code Changes

Changed:

```text
runtime/runner.py
runtime/README.md
```

Behavior after the update:

```text
remove per-loop SYMBOL_SKIPPED_NO_NEW_BAR event writes
remove per-loop signals rows where eval_status == not_due
remove per-loop SNAPSHOT_OK event writes
keep snapshot.csv as the loop summary source
keep CACHE_UPDATED only when a timeframe actually advances on a closed bar
```

The loop summary now lives in:

```text
snapshot_<YYYYMM>.csv
ALIVE_START console heartbeat
```

instead of repeated event/signal spam.

## Expected Effect

This should materially reduce daily log growth while preserving:

```text
trade truth
portfolio accept/reject decisions
state commits
error transitions
market stale/resumed transitions
GPS writes
```

The intended outcome is lower storage use without weakening status.py, GPS, or
postmortem review.
